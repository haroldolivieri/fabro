# CLAUDE.md — taps-keys-schemas

This file is the constitution for AI agents working in the `taps-keys-schemas` repo.

## What this repo is

`taps-keys-schemas` is the **single source of truth** for Skyscanner's flight pricing key schemas. It contains all 142 schema definitions (71 one-way, 71 return) in `taps_keys_schemas/schemas.json`. Both the Java `taps-keys` library and the Python port are downstream consumers.

## Immutable rules

- **Encoding rules are immutable.** The base-32 alphabet, component bit widths, and carrier offset are defined by the Java library. Do not change `bits` or `encoded_length` values in component definitions.
- **Component order matters.** The order of components in `components` arrays must exactly match the Java builder chain order. Swapping two components produces different (wrong) keys.
- **Both Java and Python consume this package.** A schema change here must be reflected in fixture regeneration from the Java library before the Python port is updated.

## Before every commit

Always run:

```bash
make validate FIXTURES_DIR=/path/to/taps-keys-fixtures
```

All 13 checks must pass. Never commit a change to `schemas.json` without validation.

## Adding a new schema

Use the add-schema skill:

```
.claude/agents/add-schema.md
```

Never edit `schemas.json` by hand without re-running validation.

## Derived fields

These fields in each schema object are deterministically derived from `prefix` and `components`. Never set them to arbitrary values:

### to_string

```
prefix + "_" + "_".join(name + DisplayName for each component)
```

Display name map: `AIRPORT→Airport`, `CITY→City`, `COUNTRY→Country`, `LOCATION→Location`, `YEARMONTH→YearMonth`, `DAY→Day`, `DIRECT→Direct`, `MARKETING_CARRIER→MarketingCarrier`

When `name` is empty (DIRECT, MARKETING_CARRIER), the part is just the display name.

### encoded_length

Sum of all component `encoded_length` values.

### open_jaw_filter

- All oneway schemas: `"NONE"`
- Return schemas: `"BOTH"` if has `(AIRPORT, "origin")` AND `(AIRPORT, "destination")`; `"ORIGIN"` if only origin airport; `"DESTINATION"` if only destination airport; `"NONE"` otherwise. Only AIRPORT type counts — CITY, COUNTRY, LOCATION do not affect this.

## Fixture files

`golden_encodings.json` and `golden_signatures.json` in the fixtures directory are generated by the Java library. Do not edit them manually. After adding a schema, regenerate fixtures from the Java library before re-running `make validate`.

## validate.py

```bash
python scripts/validate.py --fixtures-dir /path/to/taps-keys-fixtures
```

Runs 13 structural and cross-reference checks. All checks run even if one fails. Exits 0 on full pass, 1 on any failure.
